Case Study
Connected Fitness

Problem

Consumer boxing hardware couldn't accurately track punch metrics in real time without offloading computation to a remote server.

Solution

An on-device edge ML pipeline that classifies motion sensor data in real time, running entirely on Android with no network dependency.

Impact

>95% Accuracy
On-device real-time predictions

Stack

Python · TensorFlow · Keras · TensorFlow Lite · NumPy · AWS EC2 · Android

Context

The client makes a connected home boxing system with wrist and ankle wearables that capture IMU sensor data (accelerometer and gyroscope). The product's core value is real-time punch and kick analytics. The pipeline had to classify movements entirely on the Android edge device, with no server round-trip during a workout.

The solution

The core model is a CNN trained on raw IMU time-series data, classifying each movement into a punch or kick type. Collecting and manually labeling enough representative samples to cover the variability in user technique was the main constraint. A VAE was trained on the existing dataset to generate synthetic samples, expanding the training set without additional manual labeling effort.

Post-training, the model was converted to TensorFlow Lite and weight-quantized to 16-bit floating point, then deployed to the Android device where it runs via the TFLite runtime entirely on-device.

Outcome

The system classifies movements in real time at over 95% accuracy with no network dependency during workouts. Per-rep analytics became a core differentiator of the product.
