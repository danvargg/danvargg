Case Study
MedTech

Problem

Paralyzed and intubated patients in intensive care have no reliable way to communicate or interact with their environment.

Solution

An eye-tracking system using computer vision that lets patients communicate and control devices through gaze alone.

Impact

HIPAA-Compliant
On-device real-time recognition

Stack

Python · TensorFlow · Keras · CoreML · coremltools · OpenCV · dlib · mediapipe · ARKit · AWS EC2 · Weights & Biases

Context

The client builds Software-as-a-Medical-Device (SaMD) applications using their patented eye-tracking technology on standard iPad hardware. The engagement covered two phases: a communication tool for ICU patients, followed by a clinical monitoring system for neurological disorders.

Phase 1: Gaze-to-Speech

The core model was a CNN taking dual eye crops and facial landmarks as input and predicting gaze coordinates on screen. Trained using data augmentation to improve robustness to clinical environment variability. Post-training, the model was converted to CoreML with 16-bit weight quantization for on-device inference.

At deployment, a short per-user calibration sequence fine-tunes a lightweight regressor on top of the frozen feature extractor, which is pushed back to the device as a CoreML model - personalizing predictions without a full retrain cycle.

Phase 2: MS Monitoring

Phase 2 extended the system for recording eye movements during visual tests to evaluate and monitor Multiple Sclerosis progression. Training used synthetically generated video data to cover stimuli patterns. The per-user calibration pattern from Phase 1 was carried forward.

Outcome

The solution is in active use. Phase 1 enables ICU patients to communicate through gaze alone. Phase 2 provides clinicians with precise eye movement signals for neurological evaluation and longitudinal monitoring of MS patients.
